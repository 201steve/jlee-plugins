# spec-forge

> BA → 기획자 → QA 3단계 워크플로우로 PRD를 자동 생성하는 Claude 플러그인

**퍼블리셔**: jlee-plugins

---

## 무엇을 하는 플러그인인가

요구사항 텍스트를 입력하면 3개의 에이전트 역할이 순서대로 실행되어 완성된 PRD 마크다운 파일을 생성합니다.

1. **BA (Business Analyst)** — 요구사항을 8개 카테고리로 분석해 구조화된 PRD 초안 작성
2. **기획자 (Planner)** — UX/비즈니스 관점에서 초안을 보완
3. **QA** — 테스트 시나리오, 엣지케이스, 인수 기준 추가

## 스킬

### `prd` — PRD 생성

**트리거**: "PRD 만들어줘", "PRD 작성해줘", "기획서 만들어줘"

**입력 방법**:
- 채팅에 요구사항 텍스트 붙여넣기 + "PRD 만들어줘"
- 요구사항 파일 첨부 + "PRD 만들어줘"

**출력**: `PRD-{기능명}.md` 파일 (작업 폴더에 저장)

**PRD 구성 섹션**:
1. 개요 (목적, 대상 사용자, 성공 지표)
2. 사용자 스토리
3. 기능 요구사항 (인수 기준 포함)
4. 테스트 시나리오 (Given/When/Then)
5. 미결 사항
